# 중학생용 AI 비전 탐구·윤리 프로젝트
## MediaPipe × YOLO 기반 10차시 운영안

## 1. 프로그램 개요

| 구분 | 내용 |
|---|---|
| 프로그램명 | AI 비전 탐구와 윤리 프로젝트 |
| 대상 | 중등 1~3학년 권장 |
| 운영 차시 | 10차시 |
| 권장 시간 | 차시당 50분 또는 90분 블록수업 |
| 권장 팀 구성 | 3~4명 |
| 핵심 기술 | MediaPipe Hands / Face / Pose, YOLO Object Detection |
| 수업 성격 | 탐구형, 분석형, 프로젝트형 |
| 최종 산출물 | 프로젝트 기획안, 프로토타입 초안, 윤리 분석 발표 |

## 2. 운영 목표

| 구분 | 내용 |
|---|---|
| 목표 1 | MediaPipe와 YOLO의 동작 원리와 차이를 이해한다 |
| 목표 2 | 오인식, 편향, 데이터 조건, 환경 변수의 영향을 분석한다 |
| 목표 3 | AI 비전 기술의 윤리적 문제를 토론하고 기준을 세운다 |
| 목표 4 | 실제 문제 해결형 AI 프로젝트를 기획하고 시연 수준까지 발전시킨다 |

## 3. 평가 방향

| 평가 영역 | 비율 | 평가 내용 |
|---|---:|---|
| 개념 이해 | 25 | 기술 원리, 용어, 차이점 이해 |
| 실험 분석 | 25 | 오인식 원인 분석, 실험 기록의 정확성 |
| 기획 | 25 | 문제 정의, 기술 선택, 화면 흐름 설계 |
| 윤리·발표 | 25 | 윤리 판단, 질의응답, 논리적 전달 |

## 4. 차시별 운영안

### 1차시. 컴퓨터는 사람을 어떻게 볼까?

| 항목 | 내용 |
|---|---|
| 수업 목표 | 컴퓨터 비전과 AI 비전의 기본 개념을 이해한다 |
| 핵심 활동 | MediaPipe와 YOLO 데모 비교 관찰, 사람이 보는 방식과 AI가 보는 방식 비교 |
| 기술 포인트 | 랜드마크 추적과 객체 탐지의 차이 이해 |
| AI 개념 | 입력 영상, 특징 추출, 판단 결과의 흐름 |
| 윤리 포인트 | 카메라 기반 AI는 편리하지만 감시 문제를 낳을 수 있음 |
| 결과물 | AI 비전 개념 정리표 |

### 2차시. MediaPipe Hands 분석 실험

| 항목 | 내용 |
|---|---|
| 수업 목표 | 손 랜드마크 추적의 구조와 활용 가능성을 안다 |
| 핵심 활동 | 손동작별 인식 비교, 거리·각도·조명에 따른 변화 기록 |
| 기술 포인트 | 21개 손 포인트, 손모양 패턴 인식 |
| AI 개념 | 점 좌표 기반 추적과 제스처 해석 |
| 윤리 포인트 | 손동작도 데이터이며 기록·저장 여부를 구분해야 함 |
| 결과물 | 실험 관찰표 1 |

### 3차시. Face와 Pose, 어디까지 믿을 수 있을까?

| 항목 | 내용 |
|---|---|
| 수업 목표 | 얼굴과 자세 인식의 가능성과 한계를 분석한다 |
| 핵심 활동 | 표정, 시선, 자세 변화 실험 후 어떤 조건에서 흔들리는지 기록 |
| 기술 포인트 | 얼굴 랜드마크, 자세 랜드마크, 추정 기반 인식 |
| AI 개념 | 인식과 해석은 다르며, 추정 결과는 오류 가능성이 있음 |
| 윤리 포인트 | 감정 추정의 위험, 외모·표정·몸동작의 오해 문제 |
| 결과물 | 오인식 분석표 2 |

### 4차시. YOLO 객체 탐지의 원리

| 항목 | 내용 |
|---|---|
| 수업 목표 | 객체 탐지 결과의 구성 요소를 이해한다 |
| 핵심 활동 | 교실·복도·사물 이미지에서 탐지 결과 비교, 잘못된 라벨 찾기 |
| 기술 포인트 | 바운딩 박스, 클래스 라벨, 신뢰도 |
| AI 개념 | 객체 탐지는 “무엇이 어디에 있는가”를 찾는 과정 |
| 윤리 포인트 | 탐지 결과를 사실처럼 단정하면 위험할 수 있음 |
| 결과물 | 객체 탐지 결과 해석지 |

### 5차시. AI는 왜 틀리는가?

| 항목 | 내용 |
|---|---|
| 수업 목표 | 오인식 원인을 환경, 데이터, 상황 측면에서 구분한다 |
| 핵심 활동 | 가림, 배경 혼잡, 유사 물체, 빠른 움직임, 낮은 조명 실험 |
| 기술 포인트 | 입력 품질과 모델 성능의 관계 |
| AI 개념 | 학습 데이터와 실제 환경 차이, 일반화 한계 |
| 윤리 포인트 | 오류가 출석, 안전, 평가에 쓰이면 어떤 문제가 생기는지 토론 |
| 결과물 | 실패 조건 분석표 |

### 6차시. AI 윤리 토론: 편리함과 위험 사이

| 항목 | 내용 |
|---|---|
| 수업 목표 | AI 비전 기술의 사회적 영향을 다각도로 본다 |
| 핵심 활동 | 얼굴인식 출석, 교내 안전 감시, 행동 분석, 자율주행 사례 토론 |
| 기술 포인트 | 같은 기술도 적용 맥락에 따라 의미가 달라짐 |
| AI 개념 | 기술적 가능성과 사회적 정당성은 별개 |
| 윤리 포인트 | 프라이버시, 동의, 공정성, 책임 소재, 인간 감독 |
| 결과물 | 팀별 윤리 기준표 |

### 7차시. 문제를 정의하고 AI 솔루션 기획하기

| 항목 | 내용 |
|---|---|
| 수업 목표 | 실제 생활 문제에 적합한 비전 AI 프로젝트를 설계한다 |
| 핵심 활동 | 문제정의, 사용자 설정, 기술 선택, 필요 기능 정리 |
| 기술 포인트 | MediaPipe가 적합한 문제와 YOLO가 적합한 문제 구분 |
| AI 개념 | 기술이 아니라 문제 중심으로 기획하는 태도 |
| 윤리 포인트 | 누가 이익을 얻고 누가 불편을 겪는지 점검 |
| 결과물 | 프로젝트 기획서 초안 |

### 8차시. 프로토타입 설계와 화면 구성

| 항목 | 내용 |
|---|---|
| 수업 목표 | 사용 흐름과 결과 화면을 구체화한다 |
| 핵심 활동 | 시작 화면, 인식 화면, 결과 화면, 오류 안내 화면 설계 |
| 기술 포인트 | 실시간 인식 인터페이스 구성 논리 |
| AI 개념 | 사용자 경험과 인식 정확도는 함께 설계해야 함 |
| 윤리 포인트 | 촬영 안내, 저장 여부, 오인식 경고문 반드시 포함 |
| 결과물 | 와이어프레임 또는 발표용 화면 설계안 |

### 9차시. 테스트, 피드백, 개선

| 항목 | 내용 |
|---|---|
| 수업 목표 | 다른 팀의 사용 경험을 바탕으로 문제점을 개선한다 |
| 핵심 활동 | 교차 테스트, 오류 재현, 수정안 제시 |
| 기술 포인트 | 신뢰도 낮은 결과 처리, 사용자 안내 개선 |
| AI 개념 | 좋은 AI는 정확도만이 아니라 설명 가능성과 안전성도 중요 |
| 윤리 포인트 | 실패 상황을 숨기지 않고 알리는 태도 |
| 결과물 | 개선 보고서, 발표자료 |

### 10차시. AI 프로젝트 발표와 윤리 검토

| 항목 | 내용 |
|---|---|
| 수업 목표 | 프로젝트의 기술성과 윤리성을 함께 설명한다 |
| 핵심 활동 | 팀 발표, 시연, 질의응답, 상호평가 |
| 기술 포인트 | 문제-기술-한계-개선안 구조로 발표 |
| AI 개념 | AI 시스템은 완성품이 아니라 계속 점검해야 하는 도구 |
| 윤리 포인트 | 인간의 최종 판단, 책임, 규칙의 필요성 정리 |
| 결과물 | 최종 발표자료, 프로젝트 요약서, 윤리 체크리스트 |
